# 헤이홈 광고 상품 확장 검토 — 패널 스튜디오로 무엇을 할 수 있나

> **공유 대상** 마케팅 조직 · 임베디드AI엣지파트
> **작성** 2026-07-23 | **참고자료** 키즈노트 광고 상품 소개서(2026.04 개정)
> **한 줄 요약** 키즈노트가 사람 손으로 하는 "맞춤 제휴 광고" 제작을 우리는 **도구로 자동화할 준비가 거의 끝났다.** 다만 "광고를 팔고 노출하고 정산하는 시스템"은 아직 없고, 그 관문은 기술 이슈 하나에 걸려 있다.

---

## 1. 먼저, 광고 상품은 세 겹으로 되어 있다

키즈노트 소개서를 지면 목록(스플래시·배너·팝업…)으로만 읽으면 판단을 그르친다. 광고 사업은 항상 **세 겹**이고, **겹마다 담당 조직이 다르다.**

| 겹 | 하는 일 | 예시 | 우리 쪽 담당 |
|---|---|---|---|
| **3층. 운영·정산** | 광고 자리 예약, 소재 검수, 라이브, 리포트 발행, 정산 | 키즈노트: D-3 소재 접수 → D-1 검수 → D-Day 라이브 → 어드민 리포트 | 광고 영업조직 + 운영 콘솔 (**아직 없음**) |
| **2층. 노출·측정** | 누구에게 보여줄지 고르고, 몇 명이 보고 눌렀는지 센다 | 키즈노트: 지역·자녀월령·관심사 타게팅 / CPM·CPC 과금 | 광고 서버 (**아직 없음**) |
| **1층. 지면·소재** | 광고가 실릴 자리를 만들고, 소재·랜딩페이지를 제작 | 배너 이미지, 상세페이지, 이벤트 페이지 | **패널 스튜디오 ← 여기** |

**패널 스튜디오는 1층 도구다.** 이 사실 하나가 아래 모든 판단의 출발점이다.

### 키즈노트 단가표를 이 렌즈로 다시 보면

| 상품 | 단가 | 실체 |
|---|---|---|
| 인트로 PKG | 1,200만원 / 1일 | 앱 켜자마자 뜨는 전면 광고 — **앱 본체가 소유한 자리** |
| 스플래시 | 1,000만원 / 1일 | 위와 같음 |
| 메인팝업 | 1,000만원 / 1일 | 위와 같음 |
| 프리미엄 포토엔딩 | CPM 50,000원 (CTR 15%) | 아이 사진 다 넘긴 순간 — **키즈노트만의 행동 맥락** |
| **맞춤 제휴 광고** | **"협의"** · 상품설명 = **"별도 페이지 구성"** | **← 여기가 우리 도구의 자리** |

고단가 상품은 전부 **앱 본체가 가진 자리**를 파는 것이고, 우리 스튜디오가 닿는 곳은 **제휴형(별도 페이지)** 이다. 이건 축소가 아니라 **초점**이다 — 뒤에서 보겠지만 우리에게는 제휴·전환형이 더 유리하다.

## 2. 지금 우리가 이미 가진 것

패널 스튜디오는 원래 "개발 지식 없는 사람이 스마트홈 제품 화면을 웹에서 만드는 도구"로 만들어졌다. 그런데 만들다 보니 **광고 페이지 제작에 필요한 부품이 대부분 들어와 있다.**

| 이미 되는 것 | 설명 |
|---|---|
| **광고 페이지 마법사** | 5단계 안내를 따라가면 상세페이지가 나온다. 개발자 불필요 |
| **제품 검색 → 카피 자동 제안** | 사내 지식베이스(kb.goqual.com)에서 제품 사양을 끌어와 히어로 문구·혜택 포인트를 **자동으로 초안 작성** |
| **광고 부품 4종** | 큰 이미지(히어로) · 배너 · 혜택 체크리스트 · 구매 버튼 |
| **이미지 자동 변환** | 사진·GIF·영상을 올리면 앱에 맞는 경량 포맷으로 자동 변환 |
| **랜딩 URL 사전 점검** | 광고주가 준 링크가 앱 안에서 열리는지 미리 판정 |
| **실시간 미리보기** | 폰 화면 그대로 확인 |
| **회사 계정 로그인·저장** | 구글 계정(@goqual.com)으로 로그인, 작업 내용 서버 저장 |

> **비교** 키즈노트는 이걸 "키즈노트 전용 소재 기획 지원"이라는 **사람이 붙는 서비스**로 제공한다(소개서 8p). 그래서 라이브 3영업일 전까지 소재를 넘겨야 한다. **우리는 같은 일을 마케터가 직접, 그 자리에서 한다.**

## 3. 지금 못 하는 것 — 정직하게

핵심을 비유로 설명한다.

> ### 📦 지금 광고는 "앱 안에 든 인쇄물"이다
>
> 광고 이미지와 링크가 앱 화면 안에 **찍혀서** 들어간다.
>
> 다행히 **앱을 다시 출시할 필요는 없다.** 우리 스마트홈 화면(미니앱 패널)은 앱 업데이트와
> 무관하게 갱신되고, 우리는 우리 소유 클라우드(Cube)를 쓰기 때문에 외부 승인 절차도 없다.
> 화면을 바꾸는 일 자체는 우리가 알아서 할 수 있다.
>
> 그런데도 다음 셋은 그대로 막힙니다. **인쇄물의 성질** 때문이다:
> - 인쇄물은 **모두에게 똑같다** → 사람마다 다른 광고를 보여줄 수 없다
> - 인쇄물은 **아무것도 보고하지 않는다** → 몇 명이 봤는지 셀 수 없다
> - 캠페인을 켜고 끄려면 **화면을 다시 배포**해야 한다 → 마케터 혼자 못 한다
>
> ### 🚪 가야 할 곳은 "문(門)과 그 뒤의 매장"이다
>
> 화면에는 **바뀌지 않는 문 하나**만 둔다("혜택 보기"). 문을 열면 **우리 광고 서버**로 가고,
> 무엇을 보여줄지는 그 서버가 그때그때 정한다.
> - 소재·캠페인 교체 → **즉시.** 앱도 화면도 건드리지 않는다
> - 문을 지나는 순간이 곧 클릭 → **클릭·구매 전환이 바로 집계된다**
> - 서버가 "이 집은 필터 다 됐네" 하고 판단 → **사람마다 다른 광고**

### 왜 "문 뒤"인가 — 화면을 자주 바꾸면 되는 것 아닌가

화면 배포가 아무리 싸도 위 세 가지는 안 풀립니다. 광고는 **분 단위로, 캠페인 단위로,
비개발자가** 움직이는 일인데 화면 배포는 **개발·배포 행위**다. 종류가 다른 일이다.

그래서 광고를 화면 밖으로 완전히 꺼내는 게 맞다. 이 방식의 좋은 점은 **스마트홈 화면을
사실상 한 번만 손대면 끝**이라는 것이다 — 문 위치를 옮길 때 말고는 다시 건드릴 일이 없다.

### 그리고 좋은 소식 — 측정은 생각보다 빨리 된다

앞서 "몇 명이 봤는지 셀 수 없다"고 했는데, **문 뒤로 옮기면 클릭은 바로 세어집니다.**
클릭이 우리 서버에 도착하니까요. 자사몰 구매까지 이어 붙으므로 전환도 잡힌다.

| 무엇 | 광고 서버만 세우면 |
|---|---|
| 클릭 수 · 구매 전환 | ✅ 바로 |
| 노출 수(화면에 떴다) | ⚠️ 나중에 (화면이 알려줘야 함) |

**즉 성과당 과금(CPC·CPA) 상품은 바로 팔 수 있다.** 키즈노트가 혜택 탭을 클릭당 100원에
파는 바로 그 방식이다. 노출당 과금(CPM)만 나중 과제로 남습니다.

## 4. 우리의 구조적 우위 — 여기가 진짜 이야기다

규모로는 진다. 그러나 **파는 물건의 종류가 다르다.**

### 4-1. 트래픽 구조 비교

| | 키즈노트 | 헤이홈 |
|---|---|---|
| 사용자가 앱을 여는 이유 | 알림장·아이 사진 — **매일 안 볼 수 없다** (일 3.5회) | 기기 제어 — **목적이 끝나면 나간다** |
| 규모 | DAU 160만 / MAU 260만 | *확인 필요 — 자릿수가 다를 것으로 가정* |
| 체류 시간 | 사진 보는 몰입 구간 존재 | 짧음. 자동화가 잘 될수록 앱을 덜 연다 |
| 구매력 | **추정** (20~40대 여성 80%) | **검증됨** — 실제로 우리 기기를 산 사람 |

**솔직한 결론: "많은 사람에게 보여주는" 광고(CPM 볼륨)는 우리가 불리하다.** 재고(노출 수)가 얇다.

그런데 광고에는 다른 종류가 있다.

### 4-2. 우리만 가진 축 — "추정"이 아니라 "지금 필요함"

| | 키즈노트의 타게팅 재료 | 헤이홈의 타게팅 재료 |
|---|---|---|
| 무엇 | 자녀 월령, 지역, 호칭, 관심사, 알림장 키워드, 과거 결제이력 | **보유 기기(무엇을 샀는지 확정)** + **기기가 지금 보내는 신호** |
| 성격 | **추정형·과거형** — "24개월 아이 부모니까 이런 걸 좋아하겠지" | **확정형·현재형** — "이 집 공기청정기 필터 수명이 지금 끝났다" |
| 업계 용어 | 오디언스(Audience) 타게팅 | **인텐트(Intent, 구매 의도) 타게팅** |

이 차이가 결정적이다. 키즈노트가 자랑하는 최고 성과 상품(포토엔딩, CTR 15%)은 **몰입한 순간**에 광고를 끼워 넣은 것이다. 우리가 할 수 있는 것은 **필요가 실제로 발생한 순간**에 광고를 내보내는 것이다.

> **실제 사례로 이미 구현되어 있다**
> 하츠 공기청정 패널에는 "필터 교체" 기능과 "소모품 구매" 링크, 광고 배너가 **같은 화면에** 있다. 여기에 기기 신호를 연결하면 — 필터 교체 알림이 뜬 사용자에게 그 자리에서 필터를 판다. 이건 광고가 아니라 거의 **재구매 안내**다.

**광고 업계에서 인텐트 타게팅은 오디언스 타게팅보다 단가가 높다.** 검색광고(네이버·구글)가 배너광고보다 비싼 이유와 같다. 우리는 규모로 지고 **정밀도로 이기는** 포지션이다.

### 4-3. 전환이 우리 안에서 닫힌다

우리는 자사 커머스(m.hej.life)를 갖고 있다. 광고 → 클릭 → 구매가 **우리 집안에서 완결**된다. 이 뜻은:

- **첫 광고주는 외부가 아니라 우리 자신**이어야 한다.
- 측정 시스템이 아직 없어도 **매출 증가로 효과를 자체 검증**할 수 있다.
- 실패해도 외부 광고주 신뢰를 잃지 않는다. 리스크가 0에 가깝다.

키즈노트도 같은 논리로 "키즈노트 스토어 브랜드위크"를 제휴 상품에 넣어 두었다.

### 4-4. 제작 속도

| | 키즈노트 | 헤이홈(패널 스튜디오) |
|---|---|---|
| 소재 제작 | 광고주 제작 or 키즈노트 기획 지원 | **마케터가 마법사로 직접** |
| 리드타임 | **라이브 3영업일 전** 소재 마감 | 그 자리에서 |
| 카피 초안 | 사람이 작성 | **사내 KB에서 자동 제안** |

제휴 광고는 건당 커스텀 제작이라 **제작 리드타임이 곧 영업 병목**이다. 이 병목이 우리에겐 없다.

## 5. 준비 단계 — 신호등 3단계

### 🟢 1단계. 광고 페이지 공장 (2~3주 · 즉시 착수 가능)

**지금 있는 것을 정리만 하면 된다. 신규 개발 거의 없음.**

- 광고 유형별 시작 템플릿 3종 제작 — ① 리드수집형(사전예약·상담신청) ② 체험단·이벤트형 ③ 혜택·쿠폰형
- 마법사에 광고 전용 항목 추가 (캠페인 기간, 광고주명, 랜딩 URL)
- **소재 검수 체크리스트 자동 생성** — 만들면 검수 항목이 자동으로 딸려 나온다
- 랜딩 URL 사전 점검을 검수 필수 항목으로 지정

> **이 단계의 가치는 광고 매출 이전에 나온다.** 마케팅팀이 개발자 대기 없이 상세페이지·이벤트페이지를 직접 만든다. 광고 사업을 안 하기로 해도 손해가 없다.

### 🟡 2단계. 광고 서버 세우기 (1~2개월)

앞서 설명한 "문 뒤의 매장"을 만든다. 스마트홈 화면에는 **문 하나만** 두고, 나머지를 전부
별도 웹서버가 맡습니다.

- 소재·캠페인 교체 → **즉시.** 앱도 화면도 안 건드린다
- **클릭·구매 전환 집계** → 성과당 과금(CPC·CPA) 상품 판매 개시
- 광고 서버가 사람마다 다른 광고를 고른다 → 4-2의 인텐트 광고 실현
- 랜딩이 우리 페이지라 **앱 안에서 바로 열 수 있다** (외부 브라우저로 튕기지 않아 이탈이 줄어듭니다)

**첫 광고주 = 우리 자사 커머스.** 여기서 효과를 검증한 뒤 외부에 판다.

> **초판에서 바뀐 점** — 초판은 이 단계를 "알리바바 심사 허용 여부에 달린 조건부"로 적었으나
> **그 전제가 틀렸다.** 스마트홈 화면은 앱 출시와 무관하게 갱신되고, 우리는 우리 소유
> 클라우드를 쓰므로 외부 심사 게이트가 없다. 조건부 딱지를 뗍니다. 오히려 새 방식은
> 화면을 거의 안 건드려서 **리스크가 더 낮다.**

### 🔴 3단계. 광고 영업·운영 시스템 (분기 단위 · 보류)

캠페인 관리, 광고 자리 예약, 소재 검수 큐, 광고주용 리포트 화면, 정산. — **이건 패널 스튜디오가 아니라 별도 시스템**이다. 스튜디오는 그 안에 "소재 제작기"로 들어간다.

**2단계에서 실제 노출·클릭 데이터가 나온 뒤에 착수한다.** 데이터 없이 단가표부터 만드는 것은 순서가 거꾸로다. 키즈노트가 CPM 50,000원을 받을 수 있는 이유는 DAU 160만과 리포트 시스템이 **이미 있기** 때문이지, 상품 목록이 길어서가 아니다.

## 7. 마케팅 조직에 요청하는 확인 사항

1. **헤이홈 앱 MAU / DAU 실측치** — 광고 재고의 절대량이다. CPM 상품이 성립하는지가 여기서 갈린다.
2. **앱 메인화면·스플래시에 광고 자리를 낼 수 있는지** — 앱팀 소관. 가능하다면 상품 폭이 크게 넓어진다.
3. **현재 헤이홈 이용약관·개인정보 동의가 "광고 목적 활용"을 커버하는지** — 키즈노트도 이 조건(수집 고지·최대 1년·요청 시 즉시 파기)을 소개서에 별도로 명시했다. 타게팅 광고의 법적 전제다.
4. **광고 영업 담당 유무** — 도구가 있어도 파는 사람이 없으면 매출은 0이다. 1·2단계는 자사 프로모션만으로도 가치가 있지만, 3단계는 영업 조직이 전제다.

---

## 8. 결론 및 제안

**1단계는 즉시 착수를 제안한다.** 이미 대부분 만들어져 있어 추가 비용이 거의 없고, 광고 사업의 성패와 무관하게 **마케팅팀의 페이지 제작 리드타임 단축**으로 회수된다.

**2단계(광고 서버)도 착수를 제안한다.** 초판에서 걸었던 "알리바바 심사 확인" 조건은
**전제가 틀려 해소됐다.** 자사 커머스를 첫 광고주로 삼아 검증한 뒤 외부에 판다.

**3단계와 외부 광고주 판매는 2단계 실측 이후.** 다만 초판이 "측정이 없어 못 판다"고 한 것은
과했다 — 광고 서버가 서면 **클릭당·성과당 과금은 바로 팔 수 있다.** 보류 대상은
**노출당 과금(CPM)** 이며, 그건 규모와 노출 집계가 전제다.

> ### 한 문장으로
> 키즈노트를 그대로 따라가면 우리는 **규모 싸움에서 진다.** 그러나 우리에겐 키즈노트에 없는 것이 있다 — **기기가 "지금 이게 필요하다"고 직접 알려준다.** 많이 보여주는 광고 대신 **정확할 때 보여주는 광고**를 파는 것이 우리 자리이며, 패널 스튜디오는 그 자리에 이미 절반쯤 도착해 있다.
